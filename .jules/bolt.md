## 2025-02-14 - Zustand Re-render Optimization
**Learning:** Found a specific performance bottleneck where core UI components (`TheGrid`, `EvaixShell`, `FocusStrip`) were triggering unnecessary re-renders when connecting to Zustand. Specifically, inline array literals (e.g., `s.cards || []`) as selector fallbacks defeat strict equality checks because they generate new references. Additionally, executing multiple individual selectors instead of batching them using `useShallow` creates unnecessary overhead and subscribers.
**Action:** When accessing Zustand stores in React components, avoid inline array literals as fallbacks (use a globally defined `EMPTY_ARRAY`), and bundle multiple property accesses inside a single `useShallow` call.
